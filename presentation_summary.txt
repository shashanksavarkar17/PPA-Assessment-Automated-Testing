1. PROJECT OVERVIEW
------------------------------------------------------------------------
This project is an end-to-end Automated Testing and Interaction Framework
designed to validate user flows, page transitions, and interactive elements
on dynamic web-based online assessment platforms. 

Using Selenium WebDriver (Python) paired with a clean architectural design,
the framework automates complex tasks such as dynamic multi-factor OTP 
retrieval, candidate credential verification, structural assessment scanning, 
and complex answer input validation (both MCQ and Coding environments).

2. CORE ARCHITECTURAL PATTERN: PAGE OBJECT MODEL (POM)
------------------------------------------------------------------------
The codebase strictly adheres to the Page Object Model (POM) pattern. 
POM is a standard industry best practice where:
- Each webpage or structural component is represented by its own Python class.
- The locators (XPaths, CSS selectors) and action methods are contained 
  entirely within their respective Page classes.
- The main script (main.py) only interacts with the page classes via clean 
  API methods, separating high-level execution logic from UI details.
- Benefit: This guarantees that if the platform's UI layout changes, we only 
  need to modify the selectors in the specific page file rather than rewriting 
  the main execution scripts.

4. KEY TECHNICAL HIGHLIGHTS & SOLVER CAPABILITIES
------------------------------------------------------------------------
When presenting to your manager, emphasize these high-end features:

A. Multi-Tab Dynamic OTP Fetching
Instead of relying on static mock OTPs, the framework actively opens a 
secondary browser tab, navigates to a temporary email service (Yopmail),
programmatically locates the newly arrived authentication email, parses the 
one-time code, switches back to the primary workspace, and authenticates 
the session in real-time.

B. Dynamic MCQ Option Selection Algorithm
Standard click systems break when element IDs or structures change. Our 
framework implements a resilient multi-tiered matching engine:
- It first tests five diverse XPath styles mapping siblings, ancestors, 
  and label parents.
- If these fail, it initiates a DOM-tree scan, retrieving all checkbox/radio 
  elements, inspecting their parent, grandparent, and sibling text content, 
  and matching them via case-insensitive substring comparisons.

C. Code Editor Injection (Monaco / CodeMirror)
Online assessment platforms use advanced developer-grade code editors (like 
VS Code's Monaco or CodeMirror). Standard text input keys fail to input code 
into these. Our framework handles this elegantly:
- It attempts direct value injection via browser execution of the editors' 
  internal JavaScript APIs.
- If APIs are restricted, it gracefully falls back to a OS-independent
  clipboard-injection and focus-trigger key sequencing flow.

D. Dynamic Self-Healing IPC Solver Bridge
To ensure the automated run never crashes due to API key issues, billing, 
or rate limits, the framework contains an intelligent IPC (Inter-Process 
Communication) fallback:
- If direct API calls fail, the system instantly writes the current context 
  to 'pending_request.json' and enters a robust polling loop.
- Once the request is resolved via the external bridge into 
  'resolved_response.json', the main thread resumes seamlessly.

5. BUSINESS VALUE OF THIS FRAMEWORK
------------------------------------------------------------------------
- Speed & Efficiency: Automates manual candidates testing flows, cutting regression 
  testing cycles down from hours to minutes.
- Real-world Resiliency: The integration of fallbacks, multi-tiered locators, 
  and self-healing loops means the script executes reliably without manual 
  supervision.
- Robust Logging & Auditing: Every click, dynamic state change, and potential 
  exception is logged programmatically to 'automation.log', while visual 
  failures trigger automatic screenshot captures in the 'screenshots/' folder.
